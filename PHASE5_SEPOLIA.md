Environment / RPC providers
- Use Alchemy or Infura for Sepolia RPC: set `SEPOLIA_RPC_URL` to your provider URL.
  - Alchemy example: `https://eth-sepolia.g.alchemy.com/v2/<KEY>`
  - Infura example: `https://sepolia.infura.io/v3/<KEY>`
- Local development: `GANACHE_RPC_URL=http://127.0.0.1:7545`
- ENV variables used:
  - `NETWORK` (optional, default `local`)
  - `PRIVATE_KEY` (for deployments)
  - `SEPOLIA_RPC_URL` or `ALCHEMY_SEPOLIA_URL` / `INFURA_SEPOLIA_URL`
  - `CONTRACT_ADDRESS_SEPOLIA` (optional manual override)
  - `ETHERSCAN_API_KEY` (optional — enables verification)

Deploying to Sepolia (short)
- Install deps: `npm install`
- Ensure `.env` contains `PRIVATE_KEY` and `SEPOLIA_RPC_URL` and (optionally) `ETHERSCAN_API_KEY`.
- Run:

```bash
npx hardhat run scripts/deploy.js --network sepolia
```

What the deploy script does
- Deploys `ProductionCertificate` to selected network
- Writes ABI to `client/src/config/abi/ProductionCertificate.json`
- Updates `client/src/config/addresses.json` with the deployed address under the network key
- Attempts Etherscan verification if `ETHERSCAN_API_KEY` is present

Front-end integration
- The client reads `client/src/config/addresses.json` and `client/src/config/abi/ProductionCertificate.json`.
- Do NOT hardcode addresses inside components. Use `getExpectedNetwork()` from `client/src/config/networks.js`.
- Explorer links: use `getTxLink(txHash, chainId)` in `client/src/services/links.js` (already implemented).

MetaMask & chain switching
- The app includes `switchToExpectedChain()` in `client/src/services/wallet.js` which calls `wallet_switchEthereumChain` and `wallet_addEthereumChain` as fallback.
- On connect, `WalletContext` checks `isExpectedChain` and surfaces a `Wrong network` error if mismatched. The UI should show a "Switch to Sepolia" option when `isCorrectNetwork` is false.

Etherscan verification
- After deployment the script tries to run Hardhat's `verify:verify` task.
- Alternatively run manually:

```bash
npx hardhat verify --network sepolia <CONTRACT_ADDRESS>
```

Transaction explorer links
- Links are generated by `client/src/services/links.js` using `explorerBaseUrl` in `client/src/config/networks.js`.
- The deploy script populates `addresses.json` and the frontend will construct `https://sepolia.etherscan.io/tx/<hash>` links for success flows (upload, revoke, verification).

Gas fee explanation (brief)
- Gas units × gas price = fee paid in ETH.
- Deployments generally cost more gas (contract bytecode + constructor).
- Function calls that modify state cost gas (e.g., `addCertificate`, `revokeCertificate`).
- Read-only calls (view) cost 0 and do not create transactions.

Testnet vs Mainnet
- Testnet (Sepolia): used for development, no real monetary value, faster iteration.
- Mainnet: real value, requires auditing, hardware wallets, and strict secret management.

Security notes for environment variables
- Never commit private keys or `.env` to version control. Add `.env` to `.gitignore`.
- Use separate accounts for testnets and production.
- For production, use secure secret stores (GitHub Secrets, Vault, environment variables in host) and hardware wallets where possible.
